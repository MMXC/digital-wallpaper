# 资源管理模块维护说明

本文档为数字人壁纸项目 Phase 1 资源管理框架的技术维护说明，供开发者使用。

## 1. 模块API接口说明

### 1.1 resource-registry.js 模块

资源注册模块，负责资源的注册、查询、更新和删除操作。

#### registerResource(resource)

注册新资源到系统。

**参数：**
- `resource` (Object): 资源对象
  - `type` (string): 资源类型，如 'image', 'video', '3dmodel', 'audio'
  - `name` (string): 资源名称
  - `path` (string): 资源存储路径
  - `metadata` (Object, optional): 附加元数据

**返回值：**
- `Promise<Object>`: 包含资源ID的注册结果

**示例：**
```javascript
const result = await registerResource({
  type: 'image',
  name: 'avatar.png',
  path: '/storage/images/avatar.png',
  metadata: { width: 512, height: 512 }
});
console.log(result.id); // resource_xxx
```

---

#### getResource(id)

获取指定资源的详细信息。

**参数：**
- `id` (string): 资源ID

**返回值：**
- `Promise<Object>`: 资源详情对象

**示例：**
```javascript
const resource = await getResource('resource_xxx');
console.log(resource.name, resource.type);
```

---

#### listResources(filters)

查询资源列表，支持过滤条件。

**参数：**
- `filters` (Object, optional): 过滤条件
  - `type` (string): 按资源类型过滤
  - `status` (string): 按状态过滤
  - `limit` (number): 返回数量限制
  - `offset` (number): 偏移量

**返回值：**
- `Promise<Array>`: 资源数组

**示例：**
```javascript
const images = await listResources({ type: 'image', limit: 10 });
```

---

#### deleteResource(id)

删除指定资源。

**参数：**
- `id` (string): 资源ID

**返回值：**
- `Promise<boolean>`: 删除是否成功

**示例：**
```javascript
await deleteResource('resource_xxx');
```

---

#### updateResource(id, data)

更新资源信息。

**参数：**
- `id` (string): 资源ID
- `data` (Object): 要更新的数据

**返回值：**
- `Promise<Object>`: 更新后的资源对象

**示例：**
```javascript
await updateResource('resource_xxx', { name: 'new_name' });
```

---

### 1.2 resource-loader.js 模块

资源加载模块，负责资源的加载、验证和URL生成。

#### loadResource(id)

加载指定资源。

**参数：**
- `id` (string): 资源ID

**返回值：**
- `Promise<Buffer>`: 资源二进制数据

**示例：**
```javascript
const data = await loadResource('resource_xxx');
```

---

#### validateResource(path)

验证资源文件的有效性。

**参数：**
- `path` (string): 资源文件路径

**返回值：**
- `Promise<boolean>`: 资源是否有效

**示例：**
```javascript
const isValid = await validateResource('/storage/images/test.png');
```

---

#### getResourceUrl(id)

获取资源的访问URL。

**参数：**
- `id` (string): 资源ID

**返回值：**
- `Promise<string>`: 资源访问URL

**示例：**
```javascript
const url = await getResourceUrl('resource_xxx');
// 返回如: https://cdn.example.com/resources/resource_xxx
```

## 2. 代码示例

### 2.1 资源注册示例

```javascript
const { registerResource } = require('./resource-registry');

async function uploadImage(file) {
  // 假设file是上传的文件对象
  const resource = {
    type: 'image',
    name: file.originalname,
    path: await saveToStorage(file),
    metadata: {
      size: file.size,
      mimeType: file.mimetype
    }
  };
  
  const result = await registerResource(resource);
  return result.id;
}
```

### 2.2 资源加载示例

```javascript
const { loadResource, getResourceUrl } = require('./resource-loader');

async function getResourceData(id) {
  // 获取资源二进制数据
  const data = await loadResource(id);
  
  // 获取访问URL
  const url = await getResourceUrl(id);
  
  return { data, url };
}
```

### 2.3 资源查询示例

```javascript
const { listResources, getResource } = require('./resource-registry');

async function searchImages(keyword) {
  // 查询所有图片资源
  const images = await listResources({
    type: 'image',
    limit: 20
  });
  
  // 进一步过滤
  return images.filter(img => img.name.includes(keyword));
}
```

## 3. 部署/运维指南

### 3.1 环境要求

**运行时环境：**
- Node.js >= 18.0.0
- npm >= 8.0.0

**系统依赖：**
- 支持本地存储或阿里云OSS存储后端
- 建议配置CDN加速资源访问

**环境变量：**
```
RESOURCE_STORAGE_TYPE=local|oss
RESOURCE_STORAGE_PATH=/path/to/storage
OSS_ACCESS_KEY_ID=your_key
OSS_ACCESS_KEY_SECRET=your_secret
OSS_BUCKET=your_bucket
OSS_REGION=oss-cn-hangzhou
```

### 3.2 配置说明

在项目配置文件 `config/resources.js` 中进行配置：

```javascript
module.exports = {
  // 存储类型: local 或 oss
  storage: {
    type: process.env.RESOURCE_STORAGE_TYPE || 'local',
    local: {
      basePath: process.env.RESOURCE_STORAGE_PATH || './storage'
    },
    oss: {
      accessKeyId: process.env.OSS_ACCESS_KEY_ID,
      accessKeySecret: process.env.OSS_ACCESS_KEY_SECRET,
      bucket: process.env.OSS_BUCKET,
      region: process.env.OSS_REGION
    }
  },
  
  // 资源大小限制 (单位: MB)
  maxSize: {
    image: 10,
    video: 100,
    audio: 50,
    '3dmodel': 200
  },
  
  // 支持的资源类型
  allowedTypes: ['image', 'video', 'audio', '3dmodel']
};
```

### 3.3 常见问题排查

#### Q1: 资源上传失败

**可能原因：**
- 存储目录权限不足
- 磁盘空间不足
- 网络问题（OSS模式）

**排查步骤：**
1. 检查存储目录是否存在且可写
2. 检查磁盘空间
3. 检查OSS配置是否正确

#### Q2: 资源加载超时

**可能原因：**
- 资源文件过大
- 网络延迟
- 存储服务异常

**排查步骤：**
1. 检查资源大小是否超出限制
2. 检查网络连接
3. 查看存储服务日志

#### Q3: 资源URL无法访问

**可能原因：**
- CDN未配置或配置错误
- 资源已被删除
- URL过期（如果使用临时URL）

**排查步骤：**
1. 确认CDN配置正确
2. 验证资源是否存在于存储中
3. 检查URL是否在有效期内

#### Q4: 资源类型不支持

**可能原因：**
- 未在 `allowedTypes` 中配置该类型
- 类型名称拼写错误

**排查步骤：**
1. 检查配置文件中的 `allowedTypes`
2. 确认资源类型名称正确（image/video/audio/3dmodel）
